Shell

Comecei a buscar como funciona a comunicação entre o sistema físico e o usuário, na minha busca me deparei com o Kernel que é um intermediário entre o sistema operacional e os hardwares, sua arquitetura pode ser: monolitico, microkernel, híbrido, nanonúcleo, exonúcleo. O Kernel das distros linux fora escrita em C, para acessa-lo pode ser pela interface gráfica ou pelo Shell que é meu atual objeto de estudo.

O Shell é um intérprete entre o usuário e o Kernel, como demostra a imagem a baixo.

Existem vários utilitários para interpretar e administrar o sistema operacional como o bash (Linux, MacOS), CMD (Windows), PowerShell (Windows, Linux, MacOS). Escolhi estudar pelo bash, pois é o padrão do meu sistema, cada utilitário tem sua peculiaridade, tendo diferenças em alguns comandos, mas a base é a mesma.

Shell script é um arquivo que será interpretado por algum programa tipo Shell. Suas funcionalidades variam de manipulação de arquivos, execução de programas, impressão de texto, acesso a servidores, dentre outros. Sendo bem utilizado para facilitar tarefas como, por exemplo, para que eu consiga atualizar meu sistema operacional Manjaro uso um script simples "sudo pacman -Syyuu" com isso o bash irá buscar atualizações, sincronizar repositórios linux e se houver atualizações disponíveis irá atualizar.

Brincando com o Bash

Quando estou aprendendo algo sempre gosto de ver exemplos para ter noção do que as palavras tentam explicar, e nada como experimentar para abrir a mente. Há uma série chamada Mr. Robot, onde um haker utiliza de um terminal para fazer invasões, vale a pena assistir. Entender como funciona seu sistema operacional e principalmente o linux para quem pretende mexer com servidores é fundamental na formação de um bom desenvolvedor, cientista de dados ou quaisquer outra ária relacionada.

Comecei tentando entender o que era um arquivo .sh ele é composto por um Shebang (#! caminho do bash) e em seguida o script. Criei um arquivo e o chamei-o de eso.sh nele escrevi o seguinte:

#!/bin/bash

#Linhas do script
echo 'Inicio do meu env'
echo ' '
env | grep -v '[:;]' | sort

"#Linhas do script" é um comentário

Porém, me deparei com a seguinte questão, como fazer para executar esse script salvo em um arquivo .sh? Busquei como acessar um arquivo e executa-lo, e para isso tive que entender e aprender os seguintes comandos:

  • ls - Abreviação de "list", permite listar tudo que há no diretório atual.
  • cd - Abreviação de "cahnge directory", permite acessar a um dirétorio dentro do diretório atual.
  • pws - Abreviação de "print working directory", mostra o caminho completo do diretorio atual.
  • cd .. - É a junção do comando cd com ponto seguido de ponto, o que da acesso ao dirétorio pai do seu atual.
  • bash - Permite executar um script shell
